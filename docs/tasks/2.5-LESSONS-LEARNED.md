## Technical Learnings

### macOS Accessibility API
1. **Permission Model:** User must grant via System Settings → Privacy → Accessibility
2. **AXUIElement Tree:** Application → Windows → TextArea → Value
3. **Memory Management:** Core Foundation requires `CFRelease()` for all copied values
4. **CFString Conversion:** Must convert to UTF8 C string with proper size calculation

### CGO Bridging
1. **Header:** `extern "C"` prevents name mangling
2. **Frameworks:** `#cgo LDFLAGS: -framework Cocoa -framework ApplicationServices`
3. **Build Tags:** Essential for platform-specific compilation
4. **Memory Safety:** `unsafe.Slice()` for C array → Go slice conversion
5. **Cleanup:** Provide free functions for all malloc'd memory

### Go Patterns
```go
// Darwin-specific with CGO
//go:build darwin
// +build darwin
package macos

/*
#cgo LDFLAGS: -framework ApplicationServices
#include "accessibility.h"
*/
import "C"

// Non-darwin stub
//go:build !darwin
// +build !darwin
package macos

func IsAccessibilityEnabled() bool { return false }
```

## Recommendations for Task 2.6+

### Prompt Engineering
1. **Explicit tool instruction:** "YOU MUST USE file_write() TOOL CALLS"
2. **Show format example:** `file_write("path", """content""")`
3. **List exact files:** Numbered list of files to create
4. **Keep context small:** Reference files, don't include full spec

### Tier Selection
1. **L1-Coder first** for integration tasks (proven success in 2.4)
2. **L2-Coder if L1 fails** (MiniMax is cheaper, may have different behavior)
3. **L3-Coder (Claude)** for complex tasks requiring tool calls (native format)

### Orchestrator Improvements
1. **Extract code blocks** from responses when tools not executed
2. **Better error messages** for context size exceeded
3. **Auto-escalate** on repeated context errors
