## Objective

Create a `TerminalAdapter` implementation for Windows Terminal/PowerShell/PuTTY using UI Automation API via CGO bindings. This enables automatic terminal content capture on Windows.

Create C header with UI Automation COM wrapper function declarations:

```c
// Initialize UI Automation
int uia_initialize(void);
void uia_cleanup(void);

wchar_t** get_terminal_window_titles(int* count);
int* get_terminal_window_pids(int* count);

// Extract text from terminal window
wchar_t* extract_text_from_window(int pid);
wchar_t* extract_text_from_focused_terminal(void);

// Memory management
void free_wide_string_array(wchar_t** array, int count);
void free_wide_string(wchar_t* str);
```

### 2. UI Automation Implementation (`ui_automation.c`)

Implement functions using Windows UI Automation COM API:

**Key APIs:**
- `IUIAutomation::CreateCondition()` - Create tree scope conditions
- `IUIAutomation::GetRootElement()` - Get desktop element
- `IUIAutomationElement::FindAll()` - Find terminal windows
- `IUIAutomationElement::GetCurrentName()` - Get window title
- `IUIAutomationElement::GetCurrentProcessId()` - Get PID
- `ITextPattern::GetSelection()` - Extract text content

**Functions to implement:**

1. **`uia_initialize()`** - Initialize COM and UI Automation, return 0 on success
2. **`uia_is_available()`** - Check if UI Automation is available
3. **`uia_cleanup()`** - Release COM resources
4. **`get_terminal_window_titles()`** - Find Windows Terminal/PuTTY/PowerShell windows
5. **`get_terminal_window_pids()`** - Get process IDs
6. **`extract_text_from_window(pid)`** - Extract text via UI Automation text pattern
7. **`extract_text_from_focused_terminal()`** - Get focused terminal content
8. **Memory management functions**

### 4. Adapter Implementation (`adapter_windows.go`)

Implement `TerminalAdapter` interface:

```go
// Adapter implements TerminalAdapter for Windows terminals
type Adapter struct {
    config Config
}

// Name returns "windows"
func (a *Adapter) Name() string { return "windows" }

// Available checks UI Automation availability
func (a *Adapter) Available(ctx context.Context) bool {
    return IsUIAutomationAvailable()
}

// ListSessions returns detected terminal windows
func (a *Adapter) ListSessions(ctx context.Context) ([]DetectedTerminal, error)

// Capture extracts terminal content
func (a *Adapter) Capture(ctx context.Context, terminalID string) (string, error)

// Subscribe streams terminal events
func (a *Adapter) Subscribe(ctx context.Context, terminalID string) (<-chan TerminalEvent, error)

// GetDimensions returns terminal dimensions
func (a *Adapter) GetDimensions(ctx context.Context, terminalID string) (int, int, error)
```

## Implementation Details

### UI Automation Text Extraction

Terminal emulators expose content via UI Automation Text pattern:

1. Initialize COM: `CoInitializeEx(NULL, COINIT_APARTMENTTHREADED)`
2. Create UI Automation instance: `CUIAutomation8` or `IUIAutomation`
3. Get desktop element: `IUIAutomation::GetRootElement()`
4. Find terminal windows by class name or automation ID
5. Get text pattern: `IUIAutomationElement::GetCurrentPattern(UIA_TextPatternId)`
6. Extract text: `ITextRange::GetText()`

### Supported Terminals

- **Windows Terminal** - Full UI Automation support (modern)
- **PowerShell Console** - Good UI Automation support
- **Command Prompt (cmd.exe)** - Limited UI Automation support
- **PuTTY** - Basic UI Automation support
- **ConEmu** - Variable UI Automation support

### Session ID Format

```
windows-{pid}-{window-title}
Example: windows-12345-Administrator:Windows PowerShell
```

## Testing Strategy

### Unit Tests
- Test adapter Name(), Available()
- Test error handling on non-Windows

### Integration Tests (Windows only)
1. Open Windows Terminal with content
2. Create adapter: `a := NewAdapter(DefaultConfig())`
3. List sessions: `sessions := a.ListSessions(ctx)`
4. Capture: `content := a.Capture(ctx, sessions[0].ID)`
5. Verify content matches terminal

## Security Considerations

1. **COM Threading:** Must use apartment-threaded model (COINIT_APARTMENTTHREADED)
2. **Memory Management:** Properly release COM objects to avoid leaks
3. **Privacy:** Only capture terminal content, not other applications
4. **UAC:** May require elevated privileges for some terminal types
