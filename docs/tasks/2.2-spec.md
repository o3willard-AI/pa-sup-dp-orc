## 2. Description

This task implements a tmux adapter that enables PairAdmin to connect to and interact with existing tmux sessions. The adapter will use tmux CLI commands to list sessions/panes, capture pane content, and send commands to specific panes.

This matters because tmux is a widely-used terminal multiplexer in production environments. Supporting tmux allows PairAdmin to work alongside existing sysadmin workflows without disrupting active sessions, enabling true "pair administration" where AI can observe and assist in real tmux sessions.

## 5. Implementation Steps

**Step 1: Create tmux package structure**
```bash
mkdir -p internal/terminal/tmux
```
Create `internal/terminal/tmux/config.go` with `TmuxConfig` struct and `DefaultConfig()` function that returns config with 500ms poll interval and "tmux" binary path.

**Step 2: Implement CommandRunner interface**
In `internal/terminal/tmux/adapter.go`, create a `CommandRunner` interface for testability:
```go
type CommandRunner interface {
    Run(ctx context.Context, name string, args ...string) (string, error)
}
```
Create `DefaultCommandRunner` struct that wraps `exec.CommandContext`.

**Step 3: Implement TmuxAdapter struct**
Create `TmuxAdapter` with fields: `config TmuxConfig`, `connected bool`, `mu sync.RWMutex`, `cmdRunner CommandRunner`. Implement constructor:
```go
func NewTmuxAdapter(config TmuxConfig) *TmuxAdapter
```

**Step 4: Implement Connect/Disconnect methods**
```go
func (a *TmuxAdapter) Connect(ctx context.Context) error
func (a *TmuxAdapter) Disconnect(ctx context.Context) error
func (a *TmuxAdapter) IsConnected() bool
```
Connect should verify tmux is available by running `tmux -V`. Set `connected = true` on success.

**Step 5: Implement ListSessions method**
```go
func (a *TmuxAdapter) ListSessions(ctx context.Context) ([]terminal.Session, error)
```
Run `tmux list-sessions -F "#{session_id}|#{session_name}|#{session_created}"`. Parse output using parser from Step 6. Return `[]terminal.Session`.

**Step 6: Create parser utilities**
In `internal/terminal/tmux/parser.go`, create:
```go
func ParseSessionList(output string) ([]terminal.Session, error)
func ParsePaneList(output string) ([]terminal.Pane, error)
```
Handle edge cases: empty output, malformed lines, special characters in session names.

**Step 7: Implement ListPanes and CapturePane**
```go
func (a *TmuxAdapter) ListPanes(ctx context.Context, sessionID string) ([]terminal.Pane, error)
func (a *TmuxAdapter) CapturePane(ctx context.Context, paneID string) (string, error)
```
- ListPanes: Run `tmux list-panes -t {sessionID} -F "#{pane_id}|#{pane_index}|#{pane_current_command}"`
- CapturePane: Run `tmux capture-pane -p -t {paneID}` with `-S -1000` for scrollback

**Step 8: Implement SendCommand and register adapter**
```go
func (a *TmuxAdapter) SendCommand(ctx context.Context, paneID string, command string) error
```
Run `tmux send-keys -t {paneID} {command} Enter`. Then update `internal/terminal/registry.go` to register the tmux adapter factory function.

## 8. Constraints & Gotchas

**Platform Considerations:**
- tmux is Unix-only (Linux/macOS). Windows builds should exclude tmux adapter using build tags: `//go:build !windows`
- tmux binary path may vary (`/usr/bin/tmux`, `/opt/homebrew/bin/tmux`). Allow config override.

**Known Pitfalls:**
- tmux commands can hang if session is dead. Always use `CommandContext` with timeout
- `capture-pane` output may contain ANSI escape codes. Consider stripping or preserving based on config
- Session/pane IDs can change. Always refresh session list before operations
- Special characters in commands need proper escaping for `send-keys`

**Interface Contracts:**
- Do NOT modify `internal/terminal/types.go` interface definitions without Phase lead approval
- Error types must match those in `internal/terminal/errors.go`
- All methods must accept `context.Context` as first parameter

**Testing Constraints:**
- Integration tests require tmux installed. Use build tags to skip on CI without tmux
- Mock `CommandRunner` interface for unit tests to avoid tmux dependency
- Test edge cases: empty sessions, special characters, very long pane output

**Security Considerations:**
- Never pass unvalidated paneID/sessionID directly to shell commands
- Use argument arrays with `exec.Command`, not string concatenation
- Sanitize commands sent via `SendCommand` to prevent injection
