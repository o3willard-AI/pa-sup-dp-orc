## Bugs Fixed

### Bug 1: Multi-Adapter Session Deletion

**Issue:** Detector was deleting sessions from previous adapters when polling multiple adapters.

**Root Cause:** `pollAdapters()` called `updateSessions()` for each adapter individually, but `updateSessions()` deleted sessions not in the current batch.

**Fix:** Collect all sessions from all adapters first, then call `updateSessions()` once.

**Before:**
```go
func (d *Detector) pollAdapters() {
    for _, adapter := range d.adapters {
        sessions, _ := adapter.ListSessions(d.ctx)
        d.updateSessions(sessions) // BUG: deletes other adapter sessions
    }
}
```

**After:**
```go
func (d *Detector) pollAdapters() {
    var allSessions []DetectedTerminal
    for _, adapter := range d.adapters {
        sessions, _ := adapter.ListSessions(d.ctx)
        allSessions = append(allSessions, sessions...)
    }
    d.updateSessions(allSessions) // FIX: update with all sessions
}
```

**Test that exposed bug:** `TestDetector_WithMockAdapters`

### Bug 2: Race Condition in AddAdapter

**Issue:** Race condition when adding adapter while detection loop is running.

**Root Cause:** `pollAdapters()` read `d.adapters` slice without holding mutex lock.

**Fix:** Copy adapters slice under read lock before iterating.

**Before:**
```go
func (d *Detector) pollAdapters() {
    for _, adapter := range d.adapters { // RACE: no lock
        ...
    }
}
```

**After:**
```go
func (d *Detector) pollAdapters() {
    d.mu.RLock()
    adapters := make([]TerminalAdapter, len(d.adapters))
    copy(adapters, d.adapters)
    d.mu.RUnlock()

    for _, adapter := range adapters { // SAFE: copy under lock
        ...
    }
}
```

**Test that exposed bug:** `go test -race` detected data race
