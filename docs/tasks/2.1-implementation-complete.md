## Implementation Summary

### TerminalAdapter Interface
Defined the contract for terminal multiplexer implementations with 6 methods:
- `Name()` - Returns adapter name
- `Available(ctx)` - Checks if terminal is installed
- `ListSessions(ctx)` - Lists active sessions
- `Capture(ctx, id)` - Captures terminal content
- `Subscribe(ctx, id)` - Streams terminal events
- `GetDimensions(ctx, id)` - Returns rows/cols

### Data Structures
- `DetectedTerminal` - Represents a detected terminal session
- `TerminalEvent` - Event from terminal session
- `TerminalEventType` - Event type constants (5 types defined)

### Error Types
- `ErrTerminalNotFound` - Session not found
- `ErrAdapterNotAvailable` - Adapter not installed
- `ErrCaptureFailed` - Content capture failed
- `ErrSubscriptionFailed` - Event subscription failed
