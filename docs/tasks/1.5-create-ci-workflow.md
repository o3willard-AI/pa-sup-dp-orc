## Description

Create a GitHub Actions workflow file that builds the PairAdmin application on Linux, macOS, and Windows. This CI workflow runs on every commit and pull request to ensure the application builds successfully on all target platforms.

This task matters because it provides automated verification that code changes don't break the build on any platform.

---

## Inputs

**Existing Files:**
- `go.mod` - Go module definition
- `wails.json` - Wails configuration
- `frontend/package.json` - Node.js dependencies
- `scripts/install-deps.sh` - Dependency installer (from Task 1.4)

**Reference:** IMPLEMENTATION_PLAN.md Task 1.5

## Outputs

**Create:** `.github/workflows/build.yml`

**Required Configuration:**
```yaml
# Triggers: push, pull_request
# Matrix: ubuntu-latest, macos-latest, windows-latest

#   2. Setup Go 1.21+
#   3. Setup Node.js
#   4. Install dependencies (use scripts/install-deps.sh)
#   5. Build with wails build
#   6. Upload build artifacts
```

## Constraints & Gotchas

- **Use wails build** not go build directly (Wails needs frontend build step)
- **Matrix builds** - same steps for all 3 platforms
- **Artifact naming** - Include platform in artifact name for differentiation
- **Wails dependencies** - May need GTK/Webkit on Linux (use install-deps.sh)
- **Windows builds** - May need special handling for .exe extension
