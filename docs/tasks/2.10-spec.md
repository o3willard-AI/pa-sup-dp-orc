# Task 2.10: Permission Documentation (Linux/Windows)

**Phase:** 2 - Terminal Integration
**Week:** 11-12 (per implementation plan)
**Priority:** Medium
**Dependencies:** Task 2.8 (Linux AT-SPI2), Task 2.9 (Windows UI Automation)

---

## Objective

Create comprehensive documentation for enabling accessibility permissions on Linux (AT-SPI2) and Windows (UI Automation). This ensures users can troubleshoot permission issues when the terminal adapters fail to capture terminal content.

---

## Requirements

### 1. Linux Permission Guide (`docs/permissions/linux.md`)

**Topics to cover:**

1. **AT-SPI2 Overview**
   - What is AT-SPI2?
   - Why it's needed for terminal capture
   - How PairAdmin uses it

2. **Enabling Accessibility**
   - GNOME: `gsettings set org.gnome.desktop.interface accessibility true`
   - Checking AT-SPI2 daemon status
   - Starting AT-SPI2 if not running

3. **Distribution-Specific Instructions**
   - Ubuntu/Debian
   - Fedora/RHEL
   - Arch Linux

4. **Troubleshooting**
   - Common errors and solutions
   - Testing AT-SPI2 availability
   - Permission denied errors
   - Sandbox/Flatpak issues

5. **Verification Steps**
   - How to test if AT-SPI2 is working
   - Sample commands to run

### 2. Windows Permission Guide (`docs/permissions/windows.md`)

**Topics to cover:**

1. **UI Automation Overview**
   - What is UI Automation?
   - Why it's needed for terminal capture
   - How PairAdmin uses it

   - Default availability on Windows
   - Administrator privileges when needed
   - UAC considerations

3. **Terminal-Specific Requirements**
   - Windows Terminal settings
   - PowerShell Console permissions
   - PuTTY considerations

4. **Troubleshooting**
   - Common errors and solutions
   - Testing UI Automation availability
   - Access denied errors
   - Antivirus interference

5. **Verification Steps**
   - PowerShell test scripts
   - How to test if UI Automation is working

### 3. Main Permissions Index (`docs/permissions/README.md`)
