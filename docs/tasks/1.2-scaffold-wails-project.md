## Description

Use the Wails CLI to scaffold a new Wails v2 project in the current directory. This creates the basic project structure including Go backend, frontend assets, and build configuration.

This task matters because Wails provides the desktop application framework that will host the PairAdmin UI.

---

- Go 1.21+ installed (VERIFIED: go1.22.2)
- Node.js/npm (must be installed for frontend build)
- Wails CLI (must be installed)

## Implementation Steps

1. **Check if Node.js is installed**
   ```bash
   node --version
   npm --version
   ```
   If not installed, install:
   ```bash
   sudo apt install -y nodejs npm
   ```

2. **Install Wails CLI**
   ```bash
   go install github.com/wailsapp/wails/v2/cmd/wails@latest
   ```

3. **Verify Wails is installed**
   ```bash
   wails version
   ```

4. **Initialize Wails project**
   ```bash
   cd /home/sblanken/code/paa5
   wails init -n pairadmin -t svelte
   ```
   
   Note: This creates a subdirectory `pairadmin/`. We need to move contents to root.

5. **Move Wails files to project root**
   ```bash
   # Move all files from pairadmin/ subdirectory to root
   mv pairadmin/* .
   mv pairadmin/.* . 2>/dev/null || true
   rmdir pairadmin
   ```

6. **Update go.mod module path if needed**
   ```bash
   cat go.mod
   ```
   Should still be `module github.com/pairadmin/pairadmin`

7. **Verify project builds**
   ```bash
   wails build
   ```

## Constraints & Gotchas

- **Wails init creates subdirectory:** `wails init -n pairadmin` creates `./pairadmin/` folder. Must move contents to root.
- **Module path:** Ensure `go.mod` module path doesn't get overwritten by Wails init
- **Frontend template:** Using `-t svelte` template (can also use `-t vue` or `-t react` - Svelte is lighter)
- **Build dependencies:** May need `libgtk-3-dev` and `libwebkit2gtk-4.0-dev` on Linux for Wails build
- **Node version:** Node.js 16+ recommended for frontend build
