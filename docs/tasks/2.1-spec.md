## 2. Description

This task establishes the foundational interface contract for terminal adapter implementations in PairAdmin v2.0. You will create the `TerminalAdapter` interface in `internal/terminal/types.go` along with supporting data structures (`DetectedTerminal`, `TerminalEvent`) that enable the application to interact with different terminal multiplexers (tmux, screen, Windows Terminal, etc.) through a unified API.

This task matters because it defines the abstraction layer that allows PairAdmin to support multiple terminal backends without coupling the core application logic to any specific terminal implementation. All future terminal adapter implementations (Task 2.2: tmux adapter, Task 2.3: Windows adapter) must conform to this interface.

### Step 2: Define TerminalAdapter Interface
Create `internal/terminal/types.go` with the following interface:

```go
package terminal

import "context"

// TerminalAdapter defines the contract for terminal multiplexer implementations.
// All terminal backends (tmux, screen, Windows Terminal) must implement this interface.
type TerminalAdapter interface {
    // Name returns the adapter name (e.g., "tmux", "screen")
    Name() string
    
    // Available checks if the terminal multiplexer is installed and accessible
    Available(ctx context.Context) bool
    
    // ListSessions returns all active terminal sessions
    ListSessions(ctx context.Context) ([]DetectedTerminal, error)
    
    // GetSession returns a specific session by ID or name
    GetSession(ctx context.Context, sessionID string) (*DetectedTerminal, error)
    
    // Capture retrieves the current visible content from a session
    Capture(ctx context.Context, sessionID string) (string, error)
    
    // SendInput sends keystrokes/commands to a session
    SendInput(ctx context.Context, sessionID string, input string) error
    
    // Subscribe returns a channel for receiving terminal events
    Subscribe(ctx context.Context, sessionID string) (<-chan TerminalEvent, error)
    
    // Close cleans up resources and closes connections
    Close() error
}
```

// DetectedTerminal represents a discovered terminal session.
type DetectedTerminal struct {
    // ID is the unique session identifier

    // Name is the human-readable session name
    Name string `json:"name"`
    
    // Adapter is the backend adapter name (e.g., "tmux")

    Created int64 `json:"created"`

    Windows int `json:"windows"`

### Step 4: Define TerminalEvent Struct and Types
Add to `internal/terminal/types.go`:

```go
// TerminalEventType represents the type of terminal event.
type TerminalEventType int

const (
    EventUnknown TerminalEventType = iota
    EventOutput       // New output received
    EventResize       // Terminal was resized
    EventBell         // Bell/alert triggered
    EventTitleChange  // Window title changed
    EventSessionEnd   // Session terminated
)

// String returns the string representation of the event type.
func (t TerminalEventType) String() string {
    switch t {
    case EventOutput:
        return "output"
    case EventResize:
        return "resize"
    case EventBell:
        return "bell"
    case EventTitleChange:
        return "title_change"
    case EventSessionEnd:
        return "session_end"
    default:
        return "unknown"
    }
}

// TerminalEvent represents an event from a terminal session.
type TerminalEvent struct {
    // Type is the event type
    Type TerminalEventType `json:"type"`
    
    // SessionID is the source session
    SessionID string `json:"session_id"`
    
    // Timestamp is when the event occurred (Unix epoch)
    Timestamp int64 `json:"timestamp"`
    
    // Data contains event-specific payload (e.g., output text)
    Data string `json:"data,omitempty"`
    
    // Metadata contains additional event context
    Metadata map[string]string `json:"metadata,omitempty"`
}
```

### Step 5: Define Custom Error Types
Create `internal/terminal/errors.go`:

```go
package terminal

import "fmt"

// ErrorCode represents terminal operation error codes.
type ErrorCode int

const (
    ErrUnknown ErrorCode = iota
    ErrAdapterNotAvailable
    ErrSessionNotFound
    ErrCaptureFailed
    ErrSendInputFailed
    ErrSubscriptionFailed
    ErrNotSupported
)

// TerminalError represents a terminal operation error.
type TerminalError struct {
    Code    ErrorCode
    Message string
    Op      string // Operation that failed
    Err     error  // Underlying error
}

// Error implements the error interface.
func (e *TerminalError) Error() string {
    if e.Err != nil {
        return fmt.Sprintf("terminal.%s: %s: %v", e.Op, e.Message, e.Err)
    }
    return fmt.Sprintf("terminal.%s: %s", e.Op, e.Message)
}

// Unwrap returns the underlying error.
func (e *TerminalError) Unwrap() error {
    return e.Err
}

// Helper functions for creating specific errors
func ErrNotAvailable(op string) *TerminalError {
    return &TerminalError{Code: ErrAdapterNotAvailable, Op: op, Message: "adapter not available"}
}

func ErrSessionNotFound(sessionID string) *TerminalError {
    return &TerminalError{Code: ErrSessionNotFound, Op: "GetSession", Message: fmt.Sprintf("session %s not found", sessionID)}
}

func ErrCaptureFailed(op string, err error) *TerminalError {
    return &TerminalError{Code: ErrCaptureFailed, Op: op, Message: "capture failed", Err: err}
}
```

## 8. Constraints & Gotchas

### Known Pitfalls
1. **Interface Method Signatures**: Do NOT change method signatures after this task. All future adapter implementations depend on this contract. Review carefully before committing.

2. **Context Usage**: All methods that perform I/O or blocking operations MUST accept `context.Context` as the first parameter. This is critical for timeout handling and cancellation.

3. **JSON Tags**: All exported struct fields must have `json:` tags. The frontend (Wails) will serialize these to JSON for the UI.

4. **Error Wrapping**: Use `fmt.Errorf("%w", err)` or the custom `TerminalError` type with `Unwrap()` for proper error chain handling.

### Platform Considerations
- **Cross-Platform**: This is interface definition only - no platform-specific code yet. However, design with cross-platform in mind (e.g., Windows path separators, session ID formats).
- **CGO**: Avoid CGO dependencies in this package. Terminal adapters may need CGO later, but the interface layer should remain pure Go.
- **File Paths**: Use `filepath` package conventions where paths are involved (for `WorkingDir` field).

### Interface Contracts
- **DO NOT** add methods to `TerminalAdapter` without updating all existing adapter implementations
- **DO NOT** remove fields from `DetectedTerminal` or `TerminalEvent` (add optional fields instead)
- **DO** keep error types consistent with Phase 1 error handling patterns
- **DO** ensure all exported types start with capital letters (Go convention)

### Future Compatibility
- The `Metadata` field in `TerminalEvent` is a `map[string]string` to allow flexible extension without breaking changes
- `TerminalEventType` uses `iota` starting from 0 with `EventUnknown` as the zero value for safe defaults
- Consider that future adapters may have different session ID formats (tmux uses numbers/names, Windows may use GUIDs)
