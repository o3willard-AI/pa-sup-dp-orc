# Task 2.11: Integration Testing Framework

**Phase:** 2 - Terminal Integration
**Week:** 13-14
**Priority:** High
**Dependencies:** Task 2.2 (tmux), Task 2.5 (macOS), Task 2.8 (Linux), Task 2.9 (Windows)

## Objective

Create a comprehensive integration testing framework for terminal detection and capture across all platforms. This framework enables automated testing of terminal adapters and ensures consistent behavior across Linux, macOS, and Windows.

---

## Requirements

### 1. Test Helper Package (`internal/terminal/testhelpers/`)

**Purpose:** Provide shared test utilities for terminal adapter testing.

**Components:**
- `mock_terminal.go` - Mock terminal simulator for testing
- `assertions.go` - Custom test assertions for terminal content
- `fixtures.go` - Test fixtures and sample data
- `integration.go` - Integration test helpers

### 2. Terminal Adapter Tests (`internal/terminal/adapter_integration_test.go`)

**Purpose:** Test all terminal adapters through a common interface.

**Test Cases:**
- Adapter availability detection
- Session listing
- Content capture
- Event streaming
- Dimension reporting
- Cross-platform consistency

### 3. Detector Integration Tests (`internal/terminal/detector_integration_test.go`)

**Purpose:** Test the terminal detector with multiple adapters.

**Test Cases:**
- Multi-adapter registration
- Session discovery
- Event propagation
- Adapter fallback
- Concurrent access

### 4. End-to-End Tests (`internal/terminal/e2e_test.go`)

**Purpose:** Full end-to-end testing with real/simulated terminals.

**Test Cases:**
- Start terminal → Detect → Capture → Verify
- Terminal lifecycle (open/close)
- Content update detection
- Multi-terminal scenarios

### 5. Test Coverage Report (`docs/testing/coverage-report.md`)

**Purpose:** Document test coverage and gaps.

**Contents:**
- Coverage statistics per package
- Untested scenarios
- Platform-specific gaps
- Recommendations

### Integration Test Structure

```go
func TestAdapterIntegration(t *testing.T) {
    adapters := []struct {
        name     string
        adapter  TerminalAdapter
        skipReason string // for platform-specific skips
    }{
        {"tmux", tmux.NewAdapter(tmux.DefaultConfig()), ""},
        {"linux", NewAdapter(DefaultConfig()), skipIfNotLinux()},
        {"macos", macos.NewAdapter(macos.DefaultConfig()), skipIfNotDarwin()},
        {"windows", windows.NewAdapter(windows.DefaultConfig()), skipIfNotWindows()},
    }

    for _, a := range adapters {
        t.Run(a.name, func(t *testing.T) {
            if a.skipReason != "" {
                t.Skip(a.skipReason)
            }
            testAdapterInterface(t, a.adapter)
        })
    }
}
```

## Testing Strategy

### Unit Tests
- Individual adapter methods
- Mock-based testing
- Fast execution (< 1 second per test)

### Integration Tests
- Adapter interface compliance
- Cross-platform consistency
- Moderate execution (< 10 seconds per test)

### End-to-End Tests
- Real terminal interaction (when available)
- Full workflow validation
- Longer execution (< 60 seconds per test)

## CI/CD Integration

**GitHub Actions Workflow:**

```yaml
name: Terminal Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-go@v4
      - run: go test -v ./internal/terminal/...
      - run: go test -race ./internal/terminal/...
      - run: go test -coverprofile=coverage.out ./internal/terminal/...
```
