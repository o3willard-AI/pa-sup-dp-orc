## 2. Description

This task connects the terminal capture backend (implemented in tasks 2.1-2.3) with the Wails frontend UI. You will update the UI handlers to receive terminal content updates from the capture service and push them to the frontend via Wails events. The terminal preview pane (left column) will display real-time captured terminal content, and users will be able to switch between detected tmux sessions via tab selection.

This task matters because it completes the data flow from terminal detection → capture → UI display, enabling the core "Pair Administration" experience where humans can observe AI-managed terminal sessions in real-time.

```go
    captureService terminal.CaptureService
    runtime        *wails.Runtime
    currentSession string
    mu             sync.Mutex
}

func NewTerminalUIHandler(cs terminal.CaptureService, rt *wails.Runtime) *TerminalUIHandler {
    return &TerminalUIHandler{
        captureService: cs,
        runtime:        rt,
    }
}
```

### Step 3: Implement Session Subscription Method
Add method to subscribe to capture events and emit Wails events:

```go
func (h *TerminalUIHandler) StartSubscription(ctx context.Context) {
    events := h.captureService.Subscribe()
    go func() {
        for {
            select {
            case <-ctx.Done():
                return
            case event := <-events:
                h.emitContentUpdate(event)
            }
        }
    }()
}

func (h *TerminalUIHandler) emitContentUpdate(event terminal.CaptureEvent) {
    h.mu.Lock()
    defer h.mu.Unlock()
    
    payload := map[string]interface{}{
        "session_id": event.SessionID,
        "content":    event.Content,
        "timestamp":  event.Timestamp.Unix(),
    }
    h.runtime.Events.Emit(EventTerminalContentUpdate, payload)
}
```

### Step 4: Add Session Management Handlers
Add frontend-callable methods in `handlers.go`:

```go
func (h *TerminalUIHandler) GetSessions() []terminal.TerminalSession {
    return h.captureService.GetSessions()
}

func (h *TerminalUIHandler) SwitchSession(sessionID string) error {
    h.mu.Lock()
    h.currentSession = sessionID
    h.mu.Unlock()
    
    h.runtime.Events.Emit(EventTerminalSessionSwitch, map[string]interface{}{
        "session_id": sessionID,
    })
    return nil
}
```

### Step 5: Wire Up Handler in App Initialization
Modify `internal/ui/app.go` to initialize and start the terminal handler:

### Step 6: Update Frontend Store
Modify `frontend/src/stores/terminalStore.js`:

```javascript
import { Events } from '@wailsio/runtime'

export const terminalStore = {
    sessions: [],
    currentSessionId: null,
    sessionContent: {},
    
    init() {
        Events.On('terminal:session:list', (sessions) => {
            this.sessions = sessions
            if (sessions.length > 0 && !this.currentSessionId) {
                this.currentSessionId = sessions[0].ID
            }
        })
        
        Events.On('terminal:content:update', (payload) => {
            this.sessionContent[payload.session_id] = payload.content
            if (payload.session_id === this.currentSessionId) {
                this.triggerUpdate()
            }
        })
    },
    
    async loadSessions() {
        const sessions = await backend.GetSessions()
        this.sessions = sessions
    },
    
    async switchSession(sessionId) {
        await backend.SwitchSession(sessionId)
        this.currentSessionId = sessionId
    },
    
    getCurrentContent() {
        return this.sessionContent[this.currentSessionId] || ''
    }
}
```

```jsx
import { terminalStore } from '../stores/terminalStore'

export function SessionTabs() {
    const [sessions, setSessions] = useState([])
    
    useEffect(() => {
        const unsubscribe = terminalStore.subscribe(() => {
            setSessions([...terminalStore.sessions])
        })
        setSessions([...terminalStore.sessions])
        return unsubscribe
    }, [])
    
    const handleTabClick = async (sessionId) => {
        await terminalStore.switchSession(sessionId)
    }
    
    return (
        <div className="session-tabs">
            {sessions.map(session => (
                <button
                    key={session.ID}
                    className={session.ID === terminalStore.currentSessionId ? 'active' : ''}
                    onClick={() => handleTabClick(session.ID)}
                >
                    {session.Name}
                </button>
            ))}
        </div>
    )
}
```

## 8. Constraints & Gotchas

### Known Pitfalls
1. **Race Conditions**: Multiple goroutines accessing session state. Always use mutex locks in `TerminalUIHandler` methods.
2. **Event Flooding**: Terminal content updates can be frequent. Consider debouncing if performance issues arise (max 10 updates/second).
3. **Memory Leaks**: Ensure subscription goroutines exit on context cancellation to prevent leaks.
4. **Wails Runtime**: The `runtime` field must be set after `wails.Run()` initializes. Use `startup()` lifecycle method.

### Platform Considerations
- **Linux/macOS**: tmux commands work natively
- **Windows**: tmux not available. Handler should gracefully return empty session list (no errors)
- **CGO**: No CGO dependencies in this task, but verify `go build` works with `CGO_ENABLED=0`

### Interface Contracts
- Do NOT modify `terminal.CaptureService` interface (defined in task 2.3)
- Do NOT change Wails event names once defined (frontend depends on them)
- Maintain backward compatibility with existing UI handlers
- All handler methods exposed to frontend must be thread-safe

### Error Handling
- Capture service errors should be logged, not crash the UI
- Frontend should show "No sessions detected" when session list is empty
- Network/event errors should not break the main application loop
