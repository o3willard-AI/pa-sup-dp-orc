## 2. Description

Build a background service that periodically scans for active tmux sessions and detects session lifecycle events (created, closed, modified). This detector service bridges the gap between external tmux state and internal PairAdmin session management by maintaining a mapping between terminal IDs and internal session IDs.

This task matters because it enables PairAdmin to automatically discover and track terminal sessions without manual registration, providing the foundation for real-time session monitoring and AI-assisted administration.

### Step 2: Create Detector Struct

```go
type Detector struct {
    adapter      *tmux.TmuxAdapter
    sessions     map[string]*TerminalSession  // tmux_name -> session
    sessionMutex sync.RWMutex
    eventChan    chan DetectorEvent
    stopChan     chan struct{}
    pollInterval time.Duration
    running      bool
}
```

### Step 3: Implement Constructor
```go
func NewDetector(adapter *tmux.TmuxAdapter, pollInterval time.Duration) *Detector {
    return &Detector{
        adapter:      adapter,
        sessions:     make(map[string]*TerminalSession),
        eventChan:    make(chan DetectorEvent, 100),
        stopChan:     make(chan struct{}),
        pollInterval: pollInterval,
        running:      false,
    }
}
```

### Step 4: Implement Start/Stop Methods
```go
func (d *Detector) Start() error {
    if d.running {
        return ErrDetectorAlreadyRunning
    }
    d.running = true
    go d.pollLoop()
    return nil
}

func (d *Detector) Stop() {
    if !d.running {
        return
    }
    close(d.stopChan)
    d.running = false
}
```

### Step 5: Implement Poll Loop
```go
func (d *Detector) pollLoop() {
    ticker := time.NewTicker(d.pollInterval)
    defer ticker.Stop()
    
    // Initial scan
    d.scanAndEmit()
    
    for {
        select {
        case <-ticker.C:
            d.scanAndEmit()
        case <-d.stopChan:
            return
        }
    }
}
```

### Step 6: Implement Scan and Compare Logic
```go
func (d *Detector) scanAndEmit() {
    currentSessions, err := d.adapter.ListSessions()
    if err != nil {
        select {
        case d.eventChan <- DetectorEvent{Type: "error", Timestamp: time.Now()}:
        default:
        }
        return
    }
    
    d.sessionMutex.Lock()
    defer d.sessionMutex.Unlock()
    
    // Detect new sessions
    for _, session := range currentSessions {
        if _, exists := d.sessions[session.Name]; !exists {
            d.sessions[session.Name] = session
            d.emitEvent(EventCreated, session)
        }
    }
    
    // Detect closed sessions
    for name := range d.sessions {
        found := false
        for _, session := range currentSessions {
            if session.Name == name {
                found = true
                break
            }
        }
        if !found {
            d.emitEvent(EventClosed, d.sessions[name])
            delete(d.sessions, name)
        }
    }
}
```

### Step 7: Implement Event Emission Helper
```go
func (d *Detector) emitEvent(eventType EventType, session *TerminalSession) {
    event := DetectorEvent{
        Type:      eventType,
        SessionID: session.ID,
        TmuxName:  session.Name,
        Timestamp: time.Now(),
    }
    select {
    case d.eventChan <- event:
    default:
        // Channel full, drop event (non-blocking)
    }
}
```

### Step 8: Add Public Accessor Methods
```go
func (d *Detector) Events() <-chan DetectorEvent {
    return d.eventChan
}

func (d *Detector) GetSessions() []*TerminalSession {
    d.sessionMutex.RLock()
    defer d.sessionMutex.RUnlock()
    
    sessions := make([]*TerminalSession, 0, len(d.sessions))
    for _, session := range d.sessions {
        sessions = append(sessions, session)
    }
    return sessions
}

func (d *Detector) GetSessionByTmuxName(name string) (*TerminalSession, error) {
    d.sessionMutex.RLock()
    defer d.sessionMutex.RUnlock()
    
    session, exists := d.sessions[name]
    if !exists {
        return nil, ErrSessionNotFound
    }
    return session, nil
}
```

## 8. Constraints & Gotchas

### Known Pitfalls
1. **Race Conditions**: Always use `sessionMutex` when accessing the sessions map. Read operations need `RLock()`, writes need `Lock()`.
2. **Channel Blocking**: Event channel sends must be non-blocking (use `select` with `default`) to prevent detector from hanging if consumer is slow.
3. **Tmux Availability**: Detector must handle cases where tmux is not installed or not in PATH gracefully (return error on Start, don't panic).
4. **Goroutine Leaks**: Ensure `pollLoop` exits cleanly when `stopChan` is closed. Use `defer ticker.Stop()`.

### Platform Considerations
- tmux is Unix/Linux/Mac only. Windows users will get `ErrTmuxNotAvailable`.
- Poll interval should not be less than 500ms to avoid excessive tmux command overhead.
- CGO is not required for this task (pure Go + os/exec for tmux commands).

### Interface Contracts
- Do NOT modify existing `TerminalSession` struct fields (breaks Phase 1 code)
- Do NOT change `TmuxAdapter` method signatures (already used by other components)
- Event channel must remain open until Stop() is called (consumers must range until closed or use select)
- Session IDs must be consistent (same tmux name = same internal ID across detector restarts)

### Performance Notes
- Poll interval default: 2 seconds (balance between responsiveness and system load)
- Event channel buffer: 100 events (handles burst of session changes)
- Memory: Session cache should not grow unbounded (only active tmux sessions)
